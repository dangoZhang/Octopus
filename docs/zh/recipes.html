从 Goal 开始，让触手执行。

这些是 Octopus 现在最应该顺手的任务：网页端看清 Need/Feed，本地仓库跑完整产品，接入模型，收集发布证据。用户只改 Goal，内部 harness 自己供给执行。

Browser

生成 Hello World

不安装也能看核心差异。页面先显示干净大脑生成的 Need，再由浏览器触手执行并返回 Feed。

打开 docs/app.html
填入 API key
点击 Hello World 或 Draw Octopus
Local Repo

在仓库里启动完整产品

在目标项目目录运行。用户继续调整 Goal；Need、路由、Feed、记忆和 harness 信号由 agent 处理。

octopus start --open
octopus first-run "make this repo easier to use"
octopus chat "make setup clearer"
Model

接入 Codex 或兼容模型

Provider 是运行环境，不应该污染 Need。它只负责让干净大脑和触手智能真正可用。

octopus provider save codex
source .octopus/llm.env
octopus provider status
octopus first-run --live "make this repo easier to use"
Evidence

证明本地产品能跑

发布前或排查机器环境时使用。它们是观察面，第一用户路径仍然是 Goal。

octopus start --check
octopus doctor
octopus report
octopus preflight

每个场景应该证明什么

  • 主脑只表达 Need，不绑定实现。
  • 触手拥有 prompt、tool metadata、runtime code、permission、trace 和局部思考。
  • heartbeat、memory、harness 的证据能改进下一次 Feed 供给。
