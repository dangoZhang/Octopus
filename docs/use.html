Use Octopus in five minutes.

Start the whole local product, give the brain a Goal, let tentacles supply Feed, then watch Doctor, Preflight, routes, and the pixel Octopus show what happened.

Watch the v0.1.0 live demo, read the docs tutorial, or try Hello World and Draw Octopus in the browser before installing.

One local product path

1. Install and launch

cargo install --git https://github.com/dangoZhang/Octopus octopus-core --locked --bin octopus --force
octopus start --open

The launcher prepares state, editable seed tentacles, the registry, three beats, the app, and the pixel pet.

2. Give it a Goal

octopus first-run "make this repo easier to use"

The brain writes a Goal and one clean Need. A tentacle chooses how to observe, acts, returns compact Feed, and records feedback.

3. Keep steering by Goal

octopus chat "make setup clearer"
octopus goal refine "prefer small reviewable changes"
octopus brain --goal --save "tighten the objective"

User input stays on Goal. Tool choice, repair, provider routing, and harness changes stay agent-internal or observable.

4. Read the evidence

octopus doctor
octopus report
octopus preflight
octopus pet

Doctor shows readiness. Report shows capability and gaps. Preflight shows blockers. The pixel Octopus changes color from state.

What a tentacle is

A tentacle is a local nervous system for tools. It receives a Need, handles the tool work locally, checks the result, and returns compact Feed. The seed tentacles start with repo work, computer use, maintenance, repair, write-and-run execution, and visual state.

What the brain sees

The clean brain sees only Goal, Mem, Need, and Feed. It asks for verification, comparison, reproduction, memory, forgetting, recall, or execution. The tentacle handles the implementation burden.

Add a model when ready

Octopus can use Codex login, direct API keys, local OpenAI-compatible servers, or a gateway such as LiteLLM. Provider setup is runtime plumbing, so Need text stays clean.

octopus provider save codex
source .octopus/llm.env
octopus provider check
octopus first-run --live "make this repo easier to use"
octopus provider save openai
source .octopus/llm.env
export OPENAI_API_KEY=...
octopus provider check

octopus provider status shows a coverage matrix for Goal chat, clean brain, tentacle planning, and harness evolution before the detailed model slots. Codex OAuth and local models can be ready without an API key; provider check and preflight --live provide live proof.

octopus provider matrix
OCTOPUS_LOCAL_OK=1 octopus provider matrix run
octopus provider matrix check

The matrix prepares missing provider env files, runs explicitly enabled targets, and audits a reviewable record for Codex OAuth, API-key cloud, local model, and gateway providers.

When it learns

Feed traces, check history, and repair outcomes can become harness recommendations. High-risk changes stay reviewable under .octopus/ before any apply path.

octopus install swe-agent
octopus need observe README.md
octopus feedback latest partial "feed needs sharper file evidence"
octopus beat 200
octopus repair .
