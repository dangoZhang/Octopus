# Product Gap Log

Updated: 2026-06-29

## Current Shape

- Clean brain emits cognitive `Need`; it does not choose tools.
- Tentacles own LLM prompt, tool metadata, runtime code, and evolution policy.
- Product story now frames tentacles as local nervous systems: Need stays stable, implementation is replaceable, and RouteBook learns the best supply path from feedback.
- Harness stores memory, route scores, OAuth grants, installed tentacles, goal state, Feed traces, and check history.
- Harness also stores scored repair outcomes with target tentacle/candidate hints so repair sessions can become route-learning, heartbeat, and harness-evolution evidence.
- Three beats exist: heartbeat, memory compaction, and harness route/evolution recommendation.
- Pixel pet exposes Need, action, Feed, memory, harness evolution, blocked, and success states through deterministic state mapping, and CLI can export the same pixel body as SVG.
- Pixel pet state, SVG export, and file URL helpers now live in `pet.rs`; the CLI aggregation file only maps harness status to the selected pet state.
- Native desktop pet now runs as a read-only AppKit observer over `.octopus/state.json`; it shows Goal, Need bubble, action bubbles, Feed/evolution/blocked colors, and defaults to one visible pet.
- `docs/pet.html` remains a read-only HTML preview for docs/screenshots. It no longer exposes state-control buttons.
- Field-pack loading has started as `0.1.x` foundation toward `0.2.0`: Rust loads `field-packs/`, `octopus fields` can inspect and match packs, Need context receives `field_pack`, `field_mini_task`, and expected Feed, Feed traces carry the selected field, and `evolve parallel --workers 1 --open` now samples one execution slot from the eight parallel goals while opening the desktop observer. Multi-field objectives define a candidate pool; field status and recent-run fairness choose sampled slots, `--workers n` enqueues one Need per worker slot, `evolve parallel` prepares `field-mini-task` in a fresh state, automatic and manual `needs run` repair broken installed `field-mini-task` sources before Feed, and workers now record the actual trace/verifier/status after Feed returns.
- `field-mini-task` is now the generic code-as-harness execution surface for sampled field mini tasks. It records task artifacts under `.octopus/field-mini-task/`, returns honest partial Feed when field behavior is not evolved yet, and gives harness repair a concrete trajectory to improve.
- `field-mini-task/repair-templates/` is now an editable runtime-template surface. The core loads task-specific `*.pyfrag` files from the active harness or repo template fallback; all 24 field mini task templates are migrated out of Rust core, `check field-mini-task` verifies 24/24 template coverage, compiles and executes each template in a temporary session, and live runtime Feed records the executed template path. The runner no longer keeps duplicate field-specific branches; concrete field behavior belongs to the editable harness templates. Field evolution candidates now target `repair-templates/<field>/<mini>.pyfrag` instead of inserting branches into `tools/run_field_mini_task.sh`; Rust tests assert that boundary instead of locking template internals into the core. Templates are standalone `if field/mini_task` files, so the runner no longer rewrites `elif` fragments at load time.
- Direct tentacle commands now share the bundled seed resolver: `install`, `check`, `probe`, `think`, `evolve recommend/apply/score`, and provider matrix tentacle checks can find usable seed tentacles even when a user project has its own partial `tentacles/` directory. The resolver skips local same-ID seed manifests with missing entrypoints, so direct `check`/`think`/`probe`/`evolve` paths and startup can retry bundled seeds.
- Normal `chat`, `need`, and `beat` first paths now repair broken installed seed sources before saving or building the Harness, so old state files that point at incomplete local seed copies can still recover into bundled seed harnesses.
- `report` now exposes broken installed seed sources as an explicit gap and points to `beat 200`, keeping observation read-only while the heartbeat path owns repair.
